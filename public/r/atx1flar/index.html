cluster configuration management

Google 2020

Kustomize is a Go-based open-source cluster configuration management tool for kubernetes.

I founded the project, and have taken it 6500 github stars and 100 contributors so far.

In kubernetes, one declares the state of all cluster objects in files, then applies those files to the cluster, where controllers drive objects to their newly desired states. To change the cluster one edits the files and reapplies them. This creates a new problem: how to manage hundreds of variants of cluster configs. Kustomize solves this by declaring the edits themselves as kubernetes objects.

secure, distributed computing

Google 2016

Vanadium lets distributed services find each other, securely communicate, and delegate authority using public-key cryptography.

I contributed to the Go libraries and overall project organization, wrote extensive, tested tutorials, worked closely with UX research to validate our material and built demos. One such demo is volley, a distributed OpenGL app for iOS, Android, OS X, and X11 (linux) written in Go that demonstrates vanadium’s cross platform power (code).

Vanadium and kubernetes (k8s) started about the same time, approaching distributed computing from different directions. K8s is a cluster orchestration system that runs existing software in fault tolerant fashion, while Vanadium is a set of libraries and tools to build cross-platform microservices - on PC, mobile, and IoT, that securely coordinate with each other out of the box Vanadium bakes strong crypto directly into RPC and discovery, predating service mesh concepts that are add-ons to kubernetes. Google’s investment into V was halted in favor of fuchsia and k8s.

video chat as a small business platform

Google 2014

Google Helpouts was Google’s attempt to turn video chat into a small business platform.

I worked on the Android app and the Java-based REST API it talked to, arranged for consumer and provider notifications using Google infra, and did Google search “failed query” integration that presented a Helpouts onebox, and did user studies at random locations in Palo Alto to test our mobile UX.
